# ایران مرینوس (Iran Merinos)

فروشگاه آنلاین نمایشی (Demo) برای فروش پارچه فاستونی — یک پروژه‌ی **فرانت‌اند محض (Front-end only)** که با HTML, CSS و JavaScript خام (بدون فریم‌ورک و بدون بک‌اند) ساخته شده است.

> ⚠️ **توجه مهم:** این پروژه یک نمونه‌ی آماده برای نمایش / پورتفولیو است، نه یک فروشگاه واقعی و آماده‌ی پروداکشن. هیچ سرور، دیتابیس، درگاه پرداخت واقعی یا API ای در پروژه وجود ندارد. جزئیات بیشتر در بخش [محدودیت‌ها و نکات مهم](#-محدودیتها-و-نکات-مهم) آمده است.

## توضیح کوتاه پروژه

**ایران مرینوس** یک وب‌سایت چندصفحه‌ای (Multi-page) به زبان فارسی و با چیدمان راست‌به‌چپ (RTL) است که نقش فروشگاه آنلاین یک تولیدکننده‌ی پارچه فاستونی (مرینوس، ویول، گابردین، فلانل و تویید) را بازی می‌کند. کاربر می‌تواند محصولات را مشاهده، فیلتر و جست‌وجو کند، به سبد خرید اضافه کند، وارد حساب کاربری شود و حتی یک پنل مدیریت (Admin Dashboard) نمایشی را ببیند.

تمام منطق برنامه — سبد خرید، فیلترها، ورود/ثبت‌نام، پنل ادمین — **کاملاً سمت کلاینت (Client-side)** و با `localStorage` مرورگر پیاده‌سازی شده‌اند؛ هیچ ارتباطی با سرور برقرار نمی‌شود.

## ✨ امکانات (Features)

### صفحه‌ی اصلی (`index.html`)
- بخش Hero با عنوان، دکمه‌های CTA و کارت‌های معرفی انواع پارچه
- بخش «چرا ایران مرینوس؟» با چهار ردیف متن + تصویر و انیمیشن Reveal هنگام اسکرول (با `IntersectionObserver`)
- گرید محصولات ویژه (Featured Products)
- اسلایدر نظرات مشتریان به‌صورت **بی‌نهایت و خودکار** (Infinite auto-scroll) با استفاده از `requestAnimationFrame` و `ResizeObserver`
- بخش CTA پایانی و فوتر کامل با اطلاعات تماس و لینک‌های سریع

### فروشگاه (`shop.html`)
- لیست محصولات به‌صورت کارت‌های استاتیک در HTML (نه رندر داینامیک)
- **فیلتر چندگانه**: نوع پارچه، فصل، رنگ
- **اسلایدر بازه‌ی قیمت** (Range Slider دوطرفه) با پریست‌های آماده
- **جست‌وجو** در نام/توضیح محصول (همچنین از طریق پارامتر URL مثل `shop.html?q=...`)
- **مرتب‌سازی**: ارزان‌ترین، گران‌ترین، پرفروش‌ترین
- **صفحه‌بندی (Pagination)** کلاینت-ساید با ۱۲ آیتم در هر صفحه
- تغییر نمایش گرید/لیست (ذخیره‌شده در `localStorage`)
- ناوبری با کیبورد روی کارت‌های محصول

### صفحه‌ی محصول (`product.html`)
- بارگذاری **داینامیک** اطلاعات محصول از `data/products.js` بر اساس پارامتر `?id=` در URL
- گالری تصویر با تامبنیل‌های قابل‌کلیک
- انتخاب‌گر رنگ که قیمت و گالری را به‌صورت زنده آپدیت می‌کند
- نمایش تب‌های توضیحات، مشخصات فنی کامل (جدول Spec) و نظرات
- نمایش درصد تخفیف در صورت وجود `oldPrice`

### سبد خرید (`cart.html`)
- رندر آیتم‌های سبد از `localStorage` (با Sanitize کامل ورودی‌ها برای جلوگیری از XSS)
- افزایش/کاهش تعداد، حذف آیتم (با Event Delegation به‌جای `onclick` اینلاین)
- محاسبه‌ی جمع کل، هزینه‌ی ارسال (رایگان بالای ۵۰۰,۰۰۰ تومان) و تخفیف
- سیستم **کد تخفیف نمایشی** (کدهای `MERINO10` و `IRAN20`، اعمال‌شده به‌صورت کلاینت-ساید با `btoa`)
- دکمه‌ی نهایی‌سازی خرید (در حال حاضر فقط یک پیام Toast نمایش می‌دهد)

### ورود / ثبت‌نام (`login.html`)
- فرم تب‌دار ورود و ثبت‌نام با اعتبارسنجی کامل سمت کلاینت (شماره موبایل، ایمیل، رمز عبور)
- دکمه‌ی نمایش/مخفی‌سازی رمز عبور
- ذخیره‌ی Session نمایشی در `localStorage` پس از «ورود موفق»
- شناسایی کاربر ادمین با شماره/ایمیل از پیش تعیین‌شده

### پنل مدیریت (`admin.html`)
- محافظت‌شده با بررسی وجود توکن در `localStorage` (در صورت نبود توکن، ریدایرکت به صفحه‌ی ورود)
- داشبورد آماری (فروش، سفارش، مشتری، محصولات فعال) با **داده‌های ثابت نمایشی**
- ناوبری بین بخش‌های «داشبورد»، «محصولات»، «سفارشات»، «مشتریان» و «تنظیمات» بدون رفرش صفحه
- دکمه‌ی خروج که Session را پاک می‌کند

### درباره‌ی ما / تماس با ما (`about.html`, `contact.html`)
- محتوای معرفی برند
- آکاردئون سوالات متداول (FAQ)
- فرم تماس با اعتبارسنجی و شبیه‌سازی ارسال موفق (بدون ارسال واقعی)

### امکانات سراسری (در همه‌ی صفحات)
- منوی موبایل (Hamburger Menu) با بستن خودکار با کلید `Escape`
- نوار جست‌وجوی Overlay در هدر
- نشان (Badge) تعداد آیتم‌های سبد خرید با اعداد فارسی
- سیستم Toast سراسری برای نوتیفیکیشن‌ها
- مخفی/نمایان شدن هدر هنگام اسکرول

## 🛠 تکنولوژی‌های استفاده‌شده

| فناوری | کاربرد |
|---|---|
| **HTML5** | ساخت ۸ صفحه‌ی استاتیک |
| **CSS3** خام (~۵۰۰۰ خط در یک فایل) | استایل‌دهی، گرید/فلکس‌باکس، انیمیشن، Custom Properties (`:root`) |
| **JavaScript (ES6+)** خام | تمام منطق برنامه — بدون هیچ فریم‌ورکی (React/Vue/...) |
| **Vazirmatn** (Google Fonts) | فونت فارسی پروژه، از طریق `@import` در CSS بارگذاری می‌شود |
| **Web APIs** | `localStorage`، `IntersectionObserver`، `ResizeObserver`، `requestAnimationFrame`، `URLSearchParams` |
| **SVG اینلاین** | تمام آیکون‌ها به‌صورت SVG مستقیم در HTML/JS |

❌ هیچ فریم‌ورک فرانت‌اندی، هیچ Bundler (Webpack/Vite)، هیچ پکیج Node و هیچ بک‌اندی در پروژه استفاده نشده است. `package.json` در پروژه وجود ندارد.

### نکته‌ی مهم درباره‌ی منبع داده‌ی محصولات
- فایل `data/products.js` تنها منبع داده‌ی **صفحه‌ی محصول** (`product.html`) است.
- اما کارت‌های محصول در **صفحه‌ی فروشگاه (`shop.html`)** به‌صورت HTML استاتیک نوشته شده‌اند و از `data/products.js` استفاده نمی‌کنند.
- یعنی در حال حاضر **دو منبع داده‌ی جدا** برای محصولات وجود دارد که باید هماهنگ نگه داشته شوند (نکته‌ای که در کامنت‌های خود کد هم به آن اشاره شده است).

## ▶️ نحوه‌ی اجرای پروژه

### روش ۱ — باز کردن مستقیم فایل (ساده‌ترین روش)
کافی است فایل `index.html` را در مرورگر باز کنید:

```bash
open index.html      # macOS
start index.html      # Windows
xdg-open index.html    # Linux
```

> ⚠️ به دلیل سیاست‌های امنیتی برخی مرورگرها، باز کردن مستقیم فایل ممکن است باعث محدودیت‌هایی در برخی قابلیت‌ها شود؛ بنابراین استفاده از یک سرور لوکال ساده توصیه می‌شود (روش ۲).

### روش ۲ — اجرا با یک سرور لوکال (پیشنهادی)

با Python:
```bash
python3 -m http.server 8000
```
سپس در مرورگر باز کنید: `http://localhost:8000`

یا با Node.js (پکیج `serve`):
```bash
npx serve .
```

یا با افزونه‌ی **Live Server** در VS Code (روی فایل `index.html` کلیک راست بزنید و گزینه‌ی Open with Live Server را انتخاب کنید).

## 📖 راهنمای استفاده

1. از صفحه‌ی اصلی، روی «مشاهده محصولات» کلیک کنید تا به فروشگاه بروید.
2. در فروشگاه، با فیلترهای نوع پارچه، رنگ، فصل و بازه‌ی قیمت، محصول مدنظر را پیدا کنید.
3. روی هر کارت محصول کلیک کنید تا به صفحه‌ی جزئیات بروید؛ در آنجا می‌توانید رنگ را تغییر دهید و مشخصات فنی پارچه را ببینید.
4. با دکمه‌ی «افزودن» محصول را به سبد خرید اضافه کنید (آیکون سبد خرید در هدر به‌روزرسانی می‌شود).
5. وارد صفحه‌ی `cart.html` شوید تا تعداد را تغییر دهید، آیتمی را حذف کنید یا یک کد تخفیف (`MERINO10` یا `IRAN20`) وارد کنید.
6. از طریق `login.html` می‌توانید با هر شماره‌موبایل/ایمیل و رمز عبور معتبر (حداقل ۶ کاراکتر) «وارد» شوید (این فرآیند صرفاً نمایشی است).
7. برای دیدن پنل مدیریت، با شماره‌ی `09120000000` یا ایمیل `admin@iranmerinos.ir` وارد شوید؛ سپس به `admin.html` هدایت می‌شوید.

## ⚙️ تنظیمات (Configuration)

این پروژه فایل تنظیمات (`.env`، `config.json` و مشابه) ندارد. تمام «تنظیمات» مستقیماً در کد قرار دارند:

| مورد | محل تعریف |
|---|---|
| داده‌ی محصولات | `data/products.js` (آرایه‌ی `window.PRODUCTS`) |
| کلیدهای ذخیره‌سازی `localStorage` | `im_cart`, `im_user`, `im_admin_token`, `im_view` |
| کدهای تخفیف نمایشی | داخل `assets/js/cart.js` (تابع `validateCouponDemo`) |
| اطلاعات تماس / شبکه‌های اجتماعی فوتر | داخل HTML هر صفحه (بخش `<footer>`) |
| شماره/ایمیل کاربر ادمین | داخل `assets/js/login.js` (تابع `initLoginForm`) |
| فونت پروژه | داخل `assets/css/style.css` (`@import` گوگل فونت Vazirmatn) |

## 🎨 شخصی‌سازی (Customization)

### تغییر یا افزودن محصول
محصولات صفحه‌ی جزئیات را در `data/products.js` ویرایش کنید؛ هر آبجکت شامل `id`, `name`, `price`, `specs`, `colors` و غیره است. برای محصولات جدید، یک آبجکت جدید با `id` یکتا به آرایه اضافه کنید.

> توجه: همان‌طور که در بخش ساختار پروژه ذکر شد، باید کارت متناظر را در `shop.html` نیز به‌صورت دستی اضافه/ویرایش کنید تا در فروشگاه هم نمایش داده شود.

### تغییر رنگ‌ها و ظاهر کلی
رنگ‌های اصلی (سبز برند و...) به‌صورت CSS Custom Properties در ابتدای `assets/css/style.css` تعریف شده‌اند (مثلاً `--green`, `--green-dark`, `--green-light`). تغییر این متغیرها رنگ کل سایت را تغییر می‌دهد.

### تغییر فونت
خط `@import` در ابتدای `style.css` را با هر فونت دیگری از Google Fonts جایگزین کنید و مقدار `--font` در `:root` را به‌روزرسانی نمایید.

### تغییر کدهای تخفیف
در `assets/js/cart.js`، داخل تابع `validateCouponDemo`، مقادیر داخل `new Map([...])` را ویرایش کنید (کدها با `btoa()` رمزگذاری ساده شده‌اند؛ این صرفاً مبهم‌سازی است، نه امنیت واقعی).

### تغییر آستانه‌ی ارسال رایگان
مقدار `500000` در `assets/js/cart.js` (تابع `updateSummary`) و متن متناظر در `index.html` را ویرایش کنید.

### اضافه کردن صفحه‌ی جدید
از هر صفحه‌ی موجود (مثلاً `about.html`) به‌عنوان قالب کپی بگیرید، هدر/فوتر را حفظ کنید و `<link>` به `style.css` و اسکریپت‌های مشترک (`app.js`) را اضافه کنید.

## 🚧 محدودیت‌ها و نکات مهم

این بخش صادقانه محدودیت‌های فعلی پروژه را مشخص می‌کند تا توسعه‌دهنده‌ی بعدی غافلگیر نشود:

- **بدون بک‌اند واقعی**: هیچ سرور، API یا دیتابیسی وجود ندارد. تمام «داده‌ها» (سبد خرید، کاربر، توکن ادمین) فقط در `localStorage` مرورگر کاربر ذخیره می‌شوند و با پاک‌کردن کش مرورگر از بین می‌روند.
- **احراز هویت نمایشی است**: فرم ورود هیچ بررسی واقعی رمز عبور یا اطلاعات کاربر در یک پایگاه داده انجام نمی‌دهد؛ صرفاً فرمت ورودی را اعتبارسنجی می‌کند.
- **پنل ادمین قابل دور زدن است**: محافظت پنل صرفاً با بررسی وجود یک مقدار در `localStorage` انجام می‌شود؛ این روش امنیت واقعی فراهم نمی‌کند (هر کسی می‌تواند با Developer Tools مرورگر یک توکن جعلی بسازد).
- **کدهای تخفیف امن نیستند**: گرچه با `btoa()` کمی مبهم شده‌اند، اما در سمت کلاینت قابل کشف هستند؛ خود کد به این موضوع در کامنت‌ها اشاره کرده است.
- **پرداخت واقعی وجود ندارد**: دکمه‌ی «نهایی‌سازی خرید» صرفاً یک پیام Toast نشان می‌دهد.
- **دو منبع داده‌ی جدا برای محصولات**: همان‌طور که در بخش ساختار پروژه گفته شد، `shop.html` و `data/products.js` مستقل از هم هستند.
- **فرم تماس واقعاً ایمیلی ارسال نمی‌کند**: صرفاً موفقیت را شبیه‌سازی می‌کند.

## 🧯 عیب‌یابی (Troubleshooting)

| مشکل | علت احتمالی | راه‌حل |
|---|---|---|
| سبد خرید همیشه خالی نشان داده می‌شود | حالت Incognito/Private مرورگر یا غیرفعال بودن `localStorage` | از حالت عادی مرورگر استفاده کنید؛ کد دارای error boundary است اما داده ذخیره نمی‌شود |
| صفحه‌ی `admin.html` همیشه ریدایرکت می‌شود به `login.html` | توکن ادمین در `localStorage` وجود ندارد | ابتدا از `login.html` با شماره‌ی `09120000000` وارد شوید |
| فونت فارسی (Vazirmatn) لود نمی‌شود | عدم دسترسی به اینترنت (فونت از طریق CDN گوگل فونت لود می‌شود) | اتصال اینترنت را بررسی کنید یا فونت را به‌صورت لوکال در پروژه قرار دهید |
| محصول جدید در فروشگاه دیده نمی‌شود ولی در صفحه‌ی محصول هست | محصولات `shop.html` استاتیک‌اند و از `data/products.js` خوانده نمی‌شوند | کارت HTML متناظر را دستی به `shop.html` اضافه کنید |
| رفتار عجیب هنگام باز کردن مستقیم فایل (بدون سرور) | برخی امکانات مرورگر روی پروتکل `file://` محدود می‌شوند | از یک سرور لوکال استفاده کنید (به بخش [نحوه‌ی اجرا](#%EF%B8%8F-نحوهی-اجرای-پروژه) مراجعه کنید) |
| تصاویر بخش «چرا ما» نمایش داده نمی‌شوند | مسیر نسبی فایل‌ها به‌هم خورده | اطمینان حاصل کنید ساختار پوشه‌ی `assets/img/` تغییر نکرده باشد |

## 🔮 پیشنهادها برای آینده

- اتصال به یک بک‌اند واقعی (Node.js/Express, Django, Laravel و...) برای مدیریت محصولات، کاربران و سفارش‌ها
- پیاده‌سازی احراز هویت واقعی (JWT با بررسی سمت سرور) به‌جای توکن ساختگی در `localStorage`
- یکپارچه‌سازی منبع داده‌ی محصولات (حذف کارت‌های استاتیک `shop.html` و رندر داینامیک از یک منبع واحد، مثلاً API یا `products.js`)
- اتصال فرم تماس و فرآیند پرداخت به سرویس‌های واقعی (ایمیل/SMS، درگاه پرداخت ایرانی)
- اعتبارسنجی کدهای تخفیف در سمت سرور
- افزودن تست‌های خودکار (Unit/E2E)
- بهینه‌سازی تصاویر (فرمت WebP، Lazy loading بیشتر) برای بهبود سرعت بارگذاری
- استفاده از یک Bundler (Vite) برای ماژولار کردن و فشرده‌سازی کد در پروداکشن
- افزودن صفحه‌ی پروفایل کاربر و تاریخچه‌ی سفارش‌ها
